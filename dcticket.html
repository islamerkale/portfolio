DC Ticket App

2 Months
UX/UI Design
Figma, Google Analysis

DC Ticket is a mobile application designed to simplify the process of finding, contesting, and paying parking tickets in Washington DC. The app provides a user-friendly interface that helps citizens manage their parking violations efficiently.

Project Overview

The DC Ticket app was created to address the frustrations citizens face when dealing with parking tickets in Washington DC. The current system is fragmented, confusing, and time-consuming.

My role as the sole UX/UI designer involved conducting user research, creating wireframes and prototypes, and testing the solution with real users to validate the design decisions.

The Challenge

Design a mobile application that allows users to:

  • Easily find and view their parking tickets
  • Contest tickets with minimal effort
  • Make secure payments directly through the app
  • Receive notifications about ticket status

The solution needed to be accessible to all demographics, including non-tech-savvy users.

User Research

Research Methodology

To understand the pain points and needs of users dealing with parking tickets, I conducted:

  • 15 user interviews with DC residents who had received parking tickets
  • Survey of 100+ respondents to quantify common issues
  • Competitive analysis of existing ticket payment solutions
  • Contextual inquiry observing people using current systems

Key Findings

The research revealed several critical pain points:

  • Difficulty finding ticket information online
  • Confusing contestation process with unclear requirements
  • No centralized system to track multiple tickets
  • Lack of reminders leading to late fees

Persona

Meet Sarah Johnson

Based on our research, we developed Sarah as our primary persona:

Age: 32
Occupation: Marketing Manager
Tech Savviness: Comfortable with technology but prefers simple interfaces
Frustrations: Wastes hours dealing with parking tickets, misses deadlines, finds current system confusing

Sarah represents our target user - busy professionals who value their time and want a straightforward way to handle parking violations.

Information Architecture

App Structure

The information architecture was designed to prioritize the most common user tasks while making all features accessible within 2-3 taps.

Key decisions included:

  • Dashboard showing all tickets at a glance
  • Separate flows for paying vs. contesting tickets
  • Dedicated section for ticket history and receipts
  • Profile settings accessible from multiple points

User Flows

We mapped out primary user flows including:

  • New user onboarding
  • Adding a vehicle
  • Paying a ticket
  • Contesting a ticket
  • Checking contest status

Wireframes

Low-Fidelity Exploration

The wireframing phase focused on establishing the core layout and interaction patterns before committing to visual design elements.

Key considerations included:

  • Prioritizing ticket information hierarchy
  • Creating clear CTAs for primary actions
  • Ensuring form fields followed mobile best practices
  • Designing for one-handed use with thumb zones

We created multiple variations for each screen and tested them with users to identify the most intuitive layouts.

Style Guide & UI Kit

Design System

The UI style guide was created to ensure consistency across all screens and future iterations.

Color Palette:
Primary blue (#2A5CAA) was chosen for its association with trust and government, with secondary colors for status indicators (green for success, red for warnings).

Typography:
Inter was selected as the primary typeface for its excellent readability on mobile screens at various sizes.

Components:
The UI kit included reusable components like buttons, form fields, cards, and navigation elements to speed up development and maintain consistency.

Prototyping

Interactive Prototypes

Using Figma, we created interactive prototypes to test key user flows before investing in high-fidelity designs.

The prototype covered:

  • Onboarding and account creation
  • Adding a vehicle to the profile
  • Viewing ticket details
  • Initiating payment or contest
  • Uploading evidence for contestation

This allowed us to validate navigation patterns and identify usability issues early in the process.

Usability Testing

Testing Methodology

We conducted moderated usability tests with 12 participants representing our target demographics.

Participants were asked to complete key tasks while we observed:

  • Finding a specific ticket
  • Paying a ticket
  • Starting a contestation
  • Checking contest status

Key Findings

The tests revealed several areas for improvement:

  • Some users missed the contestation option initially
  • The evidence upload flow needed simplification
  • Payment confirmation wasn't prominent enough
  • Terminology needed to be more user-friendly

Refining the Design

Iterations Based on Feedback

Based on usability testing results, we made several key improvements:

1. Enhanced Visibility of Contest Option
- Added a prominent "Contest This Ticket" button alongside the payment option
- Included explanatory text about the contestation process

2. Simplified Evidence Upload
- Created a guided step-by-step process
- Added example photos showing good evidence
- Implemented progress indicators

3. Improved Payment Confirmation
- Designed a full-screen success state with clear next steps
- Added option to email or text the receipt immediately

4. Terminology Updates
- Replaced bureaucratic language with plain English
- Added tooltips explaining legal terms when necessary

Mockups

Final Visual Design

The high-fidelity mockups incorporated the DC government's visual identity while maintaining a user-friendly, modern aesthetic.

Key design elements included:

  • Clean, uncluttered interface: Ample white space and clear hierarchy
  • Visual indicators: Color-coded status badges for quick scanning
  • Accessible design: AA color contrast standards met throughout
  • Mobile-first approach: Touch targets sized for fingers, not cursors

The design system ensured consistency across all screens while allowing for necessary variations in different contexts.

Results & Takeaways

Project Outcomes

The DC Ticket app design successfully addressed the key pain points identified during research:

  • Reduced time to handle tickets by 65% in usability tests
  • Increased contestation rate by making the process more accessible
  • Improved satisfaction with 92% of testers preferring it over current options

Key Learnings

This project reinforced several important UX principles:

  • The value of early and frequent user testing
  • How small UI changes can significantly impact completion rates
  • The importance of designing for emotional states (frustration when getting a ticket)
  • Balancing government requirements with user-friendly design

Next Steps

Future enhancements could include:

  • Integration with DC's parking meter system for real-time alerts
  • Expansion to other types of municipal violations
  • Personalized recommendations based on parking habits
